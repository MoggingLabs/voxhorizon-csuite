# Secrets

Single source of truth for every credential the C-Suite touches: what it is,
where it lives, and how it rotates. No secret is ever committed; only
`infra/.env.csuite.example` (placeholders) belongs in git.

## Where secrets live

- `/home/csuite/.config/voxhorizon-csuite/csuite.env` on the VPS, chmod 600,
  owned by `csuite`. Consumed by the compose stacks via `env_file`.
- The Codex `auth.json` placed into each agent data dir
  (`/opt/voxhorizon-csuite/agents/<name>/data/auth.json`), chmod 600, owned
  `10000:10000`. This is a credential file: gitignored, never logged.

## Inventory

| Secret | Purpose | Generated by | Rotation |
|---|---|---|---|
| POSTGRES_PASSWORD | Self-hosted Supabase Postgres | bootstrap | On suspected leak |
| JWT_SECRET | Supabase JWT signing | bootstrap | On suspected leak |
| SUPABASE_SERVICE_ROLE_KEY | Server-side Supabase (Data Brain) | bootstrap (from JWT) | With JWT_SECRET |
| NEXT_PUBLIC_SUPABASE_ANON_KEY | Browser Supabase | bootstrap (from JWT) | With JWT_SECRET |
| SUPABASE_DASHBOARD_PASSWORD | Supabase Studio gate | bootstrap | Quarterly |
| CSUITE_DB_DSN (csuite_readonly) | Read-only DB for the MCP | bootstrap | Quarterly |
| MC_AUTH_PASS / MC_AUTH_SECRET | Mission Control operator gate | you | Quarterly + on leak |
| CSUITE_DISPATCHER_SECRET | Bearer: Mission Control -> dispatcher | you | Quarterly + on leak |
| Codex auth.json | OpenAI via the ChatGPT/Codex subscription | Codex/ChatGPT login | When the OAuth expires |

There is deliberately no `OPENAI_API_KEY` and no `OPENROUTER_API_KEY`. A test
(`sec_no_api_key_env`) fails the build if either appears.

## Production secret rotation (separate stack)

The research flagged a leaked production `WORKER_SHARED_SECRET`. Rotate it on the
production stack per [docs/runbooks/secret-rotation.md](docs/runbooks/secret-rotation.md)
(`python -c "import secrets; print(secrets.token_hex(64))"`, update
`/opt/voxhorizon/.env`, roll the worker + operator). It must never appear in this
repo; `sec_no_worker_shared_secret_in_repo` enforces that.

## Handling rules

- `.gitignore` blocks `.env*` (except `.env.csuite.example`), `auth.json`,
  `*.pem`, `secrets/`, `backups/`.
- `.gitleaks.toml` adds custom rules (codex token shape, JWT, `sb_` keys) and the
  secret-scan workflow runs on every PR, push, and weekly.
- Never echo a secret in a log line; the dispatcher and MCP redact.

## Security model

See [SECURITY.md](SECURITY.md) for the threat model and the control-to-test
matrix. The headline: isolation from production, a scope-limited Docker socket,
read-only data access, and human-gated irreversible actions.
